# Analysis Proof Modes

Use this layer before choosing a grammar for slides that are primarily about:

- `market-trend`
- `market-gap`
- `competitive-landscape`
- `problem-frame`
- `capability-gap`
- `benchmarking`

## Purpose

Do not start analysis pages by asking which generic analysis layout should be used.

Start by asking:

- what kind of proof is actually leading the page
- what kind of support structure exists around that proof

This stage prevents analysis slides from being forced into one chart-led default.

## Allowed Proof Modes

### `stat-led`

Use when the page is anchored by:

- one stat trio
- a few dominant numeric proofs
- a headline supported by 2-4 large quantified facts

Typical traits:

- strong numeric anchors
- no single chart needs to dominate
- support often comes from short explanations or small evidence notes

### `table-led`

Use when the page is anchored by:

- criteria tables
- comparison matrices
- structured factor comparisons
- score or ranking logic

Typical traits:

- proof comes from rows and columns rather than a chart
- comparison logic matters more than visual trend shape
- support often comes from short criteria notes or takeaway boxes

### `chart-led`

Use when the page is anchored by:

- one dominant gap chart
- one trend chart
- one comparison chart
- one visual proof that clearly deserves hero status

Typical traits:

- one visual proof block should dominate
- context and diagnosis should support the chart, not compete with it

### `diagnosis-board-led`

Use when the page is anchored by:

- a structured diagnosis board
- strengths vs constraints
- issue clusters
- capability contrasts

Typical traits:

- the page reads like an argument board
- no chart is strong enough to dominate
- multiple structured clusters carry the proof together

## Selection Rules

Apply this logic in order:

1. If the page's strongest evidence is a criteria table, comparison matrix, or structured score grid, choose `table-led`.
2. If the page's strongest evidence is one chart, one gap visual, or one dominant quantitative comparison, choose `chart-led`.
3. If the page's strongest evidence is 2-4 large stats without one chart needing to dominate, choose `stat-led`.
4. If the page's evidence is best expressed as clustered diagnosis or balanced strengths/constraints rather than one chart or one table, choose `diagnosis-board-led`.

## Non-Negotiables

- Do not choose `chart-led` just because the source contains percentages.
- Do not choose `table-led` unless the table itself carries the proof.
- Do not choose `stat-led` when a chart clearly explains the argument better.
- Do not choose `diagnosis-board-led` when one chart or one table obviously deserves to lead.

## How It Affects Grammar Choice

Use proof mode to narrow grammar and variant choice:

- `stat-led`
  - prefer hero-metric and stat-board compositions
- `table-led`
  - prefer distributed comparison boards or table-carrying variants
- `chart-led`
  - prefer variants with one dominant proof block
- `diagnosis-board-led`
  - prefer split or distributed boards over chart-centered templates
