# Analysis Terminal Bridge Rules

Use these rules when a slide is:

- analysis-led
- the final analysis slide before recommendation

This is not an ordinary analysis page.

Its job is to:

- close the analysis argument
- compress the strongest implications
- pull the viewer toward the recommendation that follows

## Purpose

The final analysis slide should feel:

- decisive
- directional
- strategically loaded
- recommendation-adjacent

It should not feel:

- neutral
- open-ended
- like one more analysis page in the middle of the deck

## Core Principle

The last analysis slide must not merely restate evidence.

It must convert evidence into strategic direction.

By the end of the page, the viewer should feel:

- the core tension is now clear
- the choice space has narrowed
- the recommendation is now the logical next move

## Terminal-Slide Construction

When a slide has terminal analysis role:

- keep one clear core diagnosis, proof, or strategic tension as the anchor
- keep only the minimum support blocks needed to make that diagnosis credible
- reserve one explicit bridge block for the `therefore` logic
- let that bridge block point forward into the recommendation, not backward into more evidence

Preferred reading logic:

1. core proof or core diagnosis
2. supporting evidence or contrast
3. strategic synthesis
4. bridge to recommendation

## Bridge Block Rules

The bridge block is not a generic summary strip.

It should:

- synthesize what the analysis means
- narrow the strategic choice
- point toward the next section

It should not:

- introduce fresh evidence
- become a second insight stack
- expand into several clauses or mini bullets
- sound like a neutral recap

Prefer a form like:

- `therefore the strategy must...`
- `this means the recommendation should...`
- `the implication is to prioritize...`

## Dominance Rules

The bridge block must be visible enough to feel consequential,
but still remain subordinate to the main proof or diagnosis anchor.

That means:

- stronger than a quiet summary strip
- weaker than the hero proof block
- cleaner and more directional than a support card

## Analysis-Proof-Mode Interaction

### `chart-led`

- let the chart still carry the page
- keep support context compact
- use the bridge block to translate the chart into a strategic move

### `table-led`

- let the table carry the comparison
- keep row/column count tight
- use the bridge block to state which option or direction the comparison now favors

### `diagnosis-board-led`

- keep one core diagnosis anchor and 2-4 compressed support clusters
- do not let every issue card compete equally
- use the bridge block to state what the diagnosis implies the recommendation must do

### `stat-led`

- let 2-4 anchor stats establish the proof
- avoid inventing a hero chart
- use the bridge block to compress those stats into one directional implication

## Copy Rules

For terminal analysis slides:

- the bridge statement should be one short directional sentence
- it may contain a small `therefore` or `must` logic
- it should sound more decisive than a normal summary strip
- it should still stay concise enough to scan instantly

## Anti-Patterns

Avoid:

- ending the analysis section on a neutral market observation
- ending on a list of issues without a directional synthesis
- making the bridge block bigger than the proof anchor
- adding extra evidence into the bridge block
- turning the bridge into a banner paragraph
