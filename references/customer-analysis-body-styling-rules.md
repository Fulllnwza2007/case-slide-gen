# Customer Analysis Body Styling Rules

Use these rules when a slide is primarily about:

- `customer-journey`
- `customer-pain`
- `customer-segmentation`
- `persona`

These rules control the look of the body, not the shell.

## Purpose

Customer-analysis slides should feel:

- human-centered
- insight-led
- structured
- polished

They should not feel like:

- a plain process slide
- an infographic poster
- a generic card grid

## Body Composition Pattern

Prefer a three-mass body when the source supports it:

- left mass: compact persona or segment context
- center mass: journey or friction logic
- right mass: insight synthesis or key implications

If the source is lighter, collapse the right mass into a compact insight callout instead of inventing empty structure.

## Color Discipline

Use a restrained consulting palette with role-based color:

- dark navy or deep blue for the main structural band, journey path, and major labels
- green only for emphasis, positive validation, or key friction highlight
- light gray or off-white for secondary surfaces and quiet cards
- avoid large saturated backgrounds behind the whole body

Do not use color as decoration only. Each color should signal role.

## Surface Design

Use body surfaces intentionally:

- persona or segment panels should feel like a clean profile card, not a poster illustration
- friction cards should feel light and structured
- one most-important friction card may be visually elevated with a stronger outline, light tint, or subtle emphasis state
- insight cards or synthesis cards should feel clean and executive, not playful

Avoid:

- glossy billboard panels
- full-bleed dark blocks unless the body truly needs a hero proof card
- ornamental gradients

## Visual Rhythm

Customer-analysis pages should create a clear reading rhythm:

- identify the person or segment
- follow the journey or decision path
- isolate the main frictions
- synthesize the key insights

The page should not flatten these into equal-weight boxes.

## Persona Cue Rules

If a persona or segment cue is present:

- keep it compact
- prioritize role, life stage, income band, need-state, and one concise motivation
- use a clean illustration or icon portrait only if it supports the profile card structure
- keep the persona block smaller than the journey hero

Do not let the persona block become the visual hero on a journey-led page.

## Friction and Insight Rules

For friction points:

- tie them spatially to the relevant journey stage whenever possible
- elevate only the most critical friction, not all of them
- keep copy short and insight-led

For insight synthesis:

- prefer one clear right-side insight column or compact insight stack
- use numbering or icon anchors if it improves readability
- keep the insight language sharper than the friction language

## Prompt Translation Notes

When converting these rules into prompts:

- ask for clean, structured persona panels rather than decorative character art
- ask for one highlighted friction state rather than several equally loud cards
- ask for a right-side insight stack when the source contains multiple synthesis points
- ask for role-based color use rather than generic colorful accents

## Anti-Patterns

Avoid:

- decorative character illustration that overwhelms the profile card
- journey tiles that all look identical in emphasis
- friction cards that feel like a generic feature grid
- insight blocks that are missing or too weak to synthesize the page
- color used as a purely aesthetic wash
